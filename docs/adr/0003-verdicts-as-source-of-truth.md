# ADR-0003: Verdicts are the source of truth; Frigate's dataset is a projection

- **Status:** Accepted
- **Date:** 2026-05-22

## Context
We need a durable record of human decisions that survives re-runs, model
changes, and Frigate upgrades. An early idea was to hand-curate inside Frigate's
`dataset/` dirs directly (even via symlinks to share images across models). That
couples the truth to Frigate's internal storage, which Frigate itself manages
(its categorize API, auto-`none` assignment, `.training_metadata.json`), and
risks drift.

## Decision
The **human verdicts** (`review/verdicts.jsonl`) are the single source of truth.
Frigate's training data is a **disposable projection** of those verdicts,
(re)generated by the commit/promote step. Review is fully local and mutable;
committing is a separate, batched push.

## Consequences
- Regenerable: wipe a model's dataset and rebuild it from verdicts.
- Enables "review loosely, fix freely, commit once" — nothing touches Frigate
  until commit; corrections before commit are free, and post-commit changes
  become reclassify/delete (commit is diff-aware).
- Avoids fighting Frigate's own dataset bookkeeping (see ADR-0004); ruled out
  in-Frigate symlinks (directory walkers skip symlinked dirs by default anyway).
- Verdicts are keyed by event/image id, so re-sourcing candidates can reuse prior
  decisions where ids match.
